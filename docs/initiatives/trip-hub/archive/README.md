# Archive

## 目标

这组文档用于承接已经结项、已经过时、或者已经被替代的旧资料。

归档不是删除，而是把它们从“当前入口”里移开，避免新成员一上来就读到过期资料。

## 哪些内容应该进入归档

### 1. 已经结项且不再作为当前主任务线跟进的工作文档

比如：

- 某条后端任务线的阶段性设计稿
- 某轮临时方案对比
- 某次集中改版的执行文档

### 2. 已被新方案替代的文档

如果新方案已经成为现行入口，旧方案应转移到归档。

### 3. 历史阶段文档

当前仓库里的 `phases/` 本质上就属于历史资料层，可视作早期归档的一部分。

## 不应该归档的内容

- 当前架构说明
- 当前内容流水线说明
- 当前仍被引用的专题方案
- 当前仍被依赖的 ADR
- 当前回归清单

## 结项后的推荐动作

当一个大块任务结项时，建议按下面顺序处理：

1. 先在 `current-status.md` 里标记为 `done`
2. 在 `timeline.md` 里补一条结项记录
3. 如果该任务对应的临时执行文档已经不再需要主入口展示，就移到 `archive/`
4. 如果该任务产出的稳定结论仍然有效，就保留一份当前有效文档，不要整份都丢进归档

## 归档原则

1. 归档的是“过程文档”，不是“有效知识”。
2. 归档后仍要保证能找到来源和上下文。
3. 归档文档名应保留原任务语义，不做模糊命名。
