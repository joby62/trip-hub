# Phase 1

## 目标

先把页面骨架改成移动端优先，让手机上“能用、好点、不卡手”。

## 范围

- 改页面布局骨架
- 改导航入口
- 改日程列表的展示方式
- 不在这一阶段接入完整图文素材

## Step By Step

1. 调整 HTML 结构
   把当前左侧 sticky 导航改成适合移动端的控制层结构。

2. 建立移动端底部导航
   至少包含 `总览`、`行程`、`图片`、`清单` 四个入口。

3. 重做第一屏
   保留路线名、时间、总路线、主按钮，删掉当前过于 desktop 的并排摘要。

4. 把日程改成摘要卡片流
   每天只显示封面位、日期、城市、一句话判断和标签。

5. 新建“当天详情”容器
   先把详情页或详情抽屉骨架立起来，后续 phase 继续灌内容。

6. 调整交互状态管理
   不再依赖一次 `renderAll()` 重绘整个长页。

## 交付物

- 一套移动端优先的页面骨架
- 底部导航和详情容器
- 可进入的日程摘要卡片列表

## 验收

- 手机上第一屏不拥挤
- 日程入口清晰
- 点击任意一天都能进入独立详情视图
- 页面不再强依赖桌面左侧导航
