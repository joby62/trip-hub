# Phase 3

## 目标

补完搜索、工具区和整体细节，让页面成为一个真正可查、可筛、可带着用的移动端旅行网页。

## 范围

- 搜索层
- 避坑筛选
- 抢票提醒
- 打包清单优化
- 动效和桌面增强

## Step By Step

1. 做全屏搜索层
   搜索范围覆盖天数、地名、景点、餐馆、原文段落和图片引用。

2. 做避坑筛选器
   标签化展示 `收费不值`、`建议绕开`、`必须提前订`、`高原提醒`。

3. 做抢票和预订工具卡
   把高铁、玉龙雪山、住宿议价等关键信息做成可快速查看的工具模块。

4. 优化打包清单
   改成更适合手机单手勾选的单列结构，支持分类折叠。

5. 做 URL 深链
   支持直达某一天、某张图或某个工具模块。

6. 做桌面增强
   在不破坏移动端骨架的前提下，把平板和桌面布局拉开。

## 交付物

- 可用的全屏搜索
- 可点的避坑和工具标签
- 更顺手的移动端清单模块
- 桌面增强版布局

## 验收

- 用户能快速搜到图片或原文
- 工具区不是静态说明，而是可操作模块
- 移动端和桌面端都成立，但移动端优先级始终最高
