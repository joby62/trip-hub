# Phase 11

## 目标

重做移动端交互层，让搜索、详情、灯箱、筛选和清单在手机上真正顺手。

这一阶段重点不是换视觉，而是把交互触达和状态切换做成手机逻辑。

## 范围

- 搜索层的手机入口、尺寸和结果操作
- 详情层 / 灯箱 / 原文跳转的手机行为
- 阶段筛选、避坑筛选、日期 rail 的手机触控表现
- 清单、工具区、景点跳转在手机上的单手可用性

## Step By Step

1. 重做搜索层
   让搜索在手机上更像全屏工具，而不是 desktop modal 缩小版。重点处理输入区、筛选区、结果区和返回逻辑。

2. 重做详情层
   重新审视日程详情和景点详情在手机上的层级，减少遮挡与过深嵌套。

3. 重做灯箱触控
   提高关闭、切图、回原文的可触达性，避免手机上按钮太小、图片被压。

4. 调整 rail 与 filter
   阶段筛选、日期 rail、避坑筛选都要以手指滑动和点按为优先，不让按钮尺寸和间距拖后腿。

5. 优化清单与工具页
   让 checklist 里的分组、折叠、勾选、重置和工具跳转都适合单手操作。

## 交付物

- 移动端搜索层重构
- 移动端详情 / 灯箱交互重构
- 适合手指点击的 rail / filter 体系
- 单手可用的 checklist / tools 交互

## 验收

- 搜索在手机上不会挡住关键内容，也不会显得拥挤。
- 详情层与灯箱的关闭、切换、跳转都清晰可达。
- rail、filter、清单操作不再显得“小、挤、难点”。
- 手机主流程可以单手完成。
