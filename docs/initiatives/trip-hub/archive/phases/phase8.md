# Phase 8

## 目标

把所有跨视图能力收口，做到上线级完成度。

这一阶段是最终整合与精修阶段，重点不再是新增大结构，而是把全站体验、搜索、深链、性能和一致性收口。

## 范围

- 搜索跨视图升级：天 / 景点 / 图片 / 原文 / 工具
- 深链升级：支持 `view/day/attraction/image/tool`
- 桌面增强
- 动效、状态切换、滚动体验、性能优化
- 最后一轮内容校对，确保无图片遗漏、无原文缺失

## Step By Step

1. 升级搜索目标
   搜索结果应覆盖 `day / attraction / image / source / tool` 五类目标。

2. 升级深链体系
   在当前 `day / image / tool` 基础上，扩展到 `view / attraction` 级别。

3. 做跨视图状态整合
   确保从总览、行程、景点、清单之间切换时，状态、滚动、搜索和高亮逻辑一致。

4. 做桌面增强
   在不破坏移动端优先的前提下，把平板和桌面版布局拉开。

5. 做动效和微交互收口
   包括导航过渡、图文切换、灯箱、章节进入和视图切换的体验优化。

6. 做性能和完整性校验
   校查首屏资源、图片组织、原文可达性、深链可达性和搜索可达性。

7. 做最终内容校对
   确保无图片遗漏、无原文缺失、无功能回退。

## 交付物

- 一套完整可交付的 editorial trip 网站
- 升级后的全站搜索与深链
- 完整桌面增强
- 最终整合后的视觉、结构、功能、数据层

## 验收

- 视觉、结构、功能、数据层全部统一。
- 搜索、深链、图片、原文、工具都已跨视图整合。
- 没有图片遗漏，没有原文缺失，没有功能回退。
