# Phase 13

## 目标

拆前端入口与配置层，建立可继续演进的模块化骨架，但不改变当前页面行为。

这一阶段重点是把“所有东西都堆在一个脚本里”的状态先拆开，让后续 phase 有稳定落点。

## 范围

- 新增 `static/guide/js/` 目录作为前端模块入口
- 切换 `yunnan.html` 到 ES module 入口
- 把内容常量、路径常量和页面配置从主脚本抽离
- 保持现有路由、搜索、详情、灯箱、清单、深链行为不变

## Step By Step

1. 收拢前端入口
   把当前单文件脚本迁到 `static/guide/js/main.js`，让 HTML 不再直接依赖根目录大脚本。

2. 抽离配置层
   把 `phaseOptions`、`dayData`、`bookingTimeline`、`SEARCH_FILTERS`、`VIEW_OPTIONS` 等稳定配置移到独立模块。

3. 收缩主脚本职责
   让 `main.js` 只负责状态、渲染调度、事件绑定和数据加载，不再混放大段内容定义。

4. 保持行为不变
   本阶段不重写页面交互，不改视觉，不调整数据结构，只做安全搬家。

5. 同步文档入口
   更新 README 与 source material 文档，让仓库入口和代码结构保持一致。

## 交付物

- `static/guide/js/main.js` 模块入口
- 独立的前端配置模块
- 更新后的页面脚本引用
- 同步后的文档入口说明

## 验收

- 页面仍然能从 `/guides/yunnan` 正常加载。
- 主交互行为不变。
- 前端入口不再是单个根目录脚本。
- 后续 phase 可以在 `js/` 目录下继续拆分，而不是回到单文件模式。
