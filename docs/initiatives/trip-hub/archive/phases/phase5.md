# Phase 5

## 目标

把“所有原文 + 所有图片”组织成真正可支撑新站的信息层，为后续总览页、行程页、景点页和清单页提供统一数据来源。

这一阶段不重点改视觉，而是把数据和素材组织从临时映射升级为稳定结构。

## 范围

- 给所有图片建立 `day + attraction/theme` 双归属
- 给所有原文段落建立 `day + block + optional attraction` 归属
- 新增统一数据层，替代现在偏临时的散映射
- 确保没有任何图片和段落孤立

## Step By Step

1. 盘点当前素材结构
   基于 `document.md`、`manifest.json`、`day-map.json` 盘点现有图片、段落和引用关系。

2. 给所有图片做双归属
   每张图都至少要归到一个 `day`，并且归到一个 `attraction` 或 `theme`。

3. 给原文段落做归类
   在不改变原文完整顺序的前提下，为段落增加天数、章节块和景点归属。

4. 新增统一数据层
   建立一层统一数据结构，把 `trip / days / attractions / media / notes / tools` 汇总起来。

5. 做孤立内容校验
   检查不存在无归属图片、无归属段落、只在代码里可达的原文块。

6. 调整前端消费入口
   让后续视图改造可以从统一数据层读取，而不是继续依赖分散的临时映射。

## 交付物

- 全量图文映射完成
- 统一数据层完成
- 图片和原文都可按“天”或“景点/主题”组织

## 验收

- 没有孤立图片
- 没有孤立原文段落
- 后续页面可以按“天”或“景点”两种逻辑组织内容
