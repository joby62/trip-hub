# Phase 10

## 目标

重做移动端首屏、卡片流和图片比例，让页面从“压缩的 desktop 视觉”转成“手机可读的内容流”。

这一阶段重点是版式和图像，不动功能删减。

## 范围

- 重做 Hero 的移动端版本
- 收敛移动端卡片的列数与密度
- 为不同内容场景重新定义图片比例
- 把 overview / itinerary / attractions / checklist 的手机正文全部调成单列优先

## Step By Step

1. 重做 Hero
   去掉手机端的重叠式 desktop hero 逻辑，改成更短、更稳的移动端首屏：主图、标题、摘要、CTA 依次下排。

2. 调整图像比例
   按内容场景拆分：
   - Hero 图
   - 景点卡图
   - 行程章节图
   - 灯箱图
   避免统一 `cover` 导致手机裁切失真。

3. 收敛移动端卡片为单列
   重点处理 `hero-highlights`、`trip-facts`、`overview-tools`、景点说明卡和工具卡。

4. 压缩首屏信息量
   手机上首屏只保留最关键的 trip framing，不让高亮卡、统计卡、主图在 390px 宽度里抢空间。

5. 统一正文节奏
   确保标题层级、段落行长、按钮密度、卡片间距都先服务手机阅读，再向上增强。

## 交付物

- 移动端新 Hero
- 一套按场景区分的图片比例规则
- overview / itinerary / attractions / checklist 的手机单列正文流
- 不再挤压的首屏和卡片栅格

## 验收

- 手机首屏不再出现大图压文案、文案压按钮、卡片压卡片。
- 主要内容区在手机上以单列阅读为主。
- 图片裁切在 Hero、章节图、景点卡、灯箱中各自合理。
- 页面看起来像“为手机设计”，不是“桌面压缩版”。
