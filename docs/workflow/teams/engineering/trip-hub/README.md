# Trip Hub Engineering Program

这是当前仓库的 owner / worker 执行骨架。

## Default Operating Mode

- `Owner`
  - 默认由 `Tech Lead` 承担
- `Worker A`
  - 默认偏内容生产和 JSON 发布线
- `Worker B`
  - 默认偏社区互动和鉴权审核线
- `Worker C`
  - 默认偏移动端 adoption、回归和集成验证线

这些只是默认倾向，不是永久绑定。
每轮由 owner 重新切分。

## Layout

- `owner/`
  - 长期 owner handoff
- `workers/`
  - Worker A / B / C 长期 handoff
- `specialists/`
  - Tech Lead 管理直属下属的 dispatch / report 规则

## Closure Discipline

一轮重要任务关闭时，owner 必须负责：

- gate conclusion
- initiative truth 是否需要补 `record / review`
- `NOW / DOC_INDEX / TIMELINE`
