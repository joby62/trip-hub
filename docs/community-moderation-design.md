# Community Moderation Design

## 目标

这份文档定义景点评价页的评论审核、滥用控制和风控边界。

它要解决的问题不是“做一个复杂社区系统”，而是先保证：

1. 评论可读可发
2. 垃圾内容不会立刻失控
3. 图片评论不会成为风险入口
4. 点赞 / UP / 踩不被随意刷爆

## 设计原则

1. 第一版先做最小可控，不做重社区产品。
2. 评论和评论图优先管住，反应类操作优先限流防刷。
3. 审核状态必须写回数据库，不靠人工口头记录。
4. 审核策略和展示策略分开。

## 风险面拆解

### 1. 文本评论风险

- 广告
- 联系方式导流
- 侮辱和攻击内容
- 低质量刷屏

### 2. 图片评论风险

- 不相关图片
- 广告图
- 违规图
- 重复刷图

### 3. 行为风险

- 批量刷点赞
- 批量刷 UP / 踩
- 高频重复评论
- 多账号短时间灌水

## 第一版审核策略

### 文本评论

建议策略：

- 先过基础规则
- 命中高风险规则直接拒绝
- 普通文本评论默认可见
- 可疑文本进入 `pending_review`

### 图片评论

建议策略：

- 带图评论默认 `pending_review`
- 审核通过后再进入 `visible`

这是第一版最稳的做法。

原因：

- 图片审核成本高
- 风险比纯文本大
- 中国内容场景下，图片是最容易出问题的入口

### 点赞 / UP / 踩

这类操作不走人工审核，但要走：

- 登录限制
- 用户频控
- 点位频控
- 设备 / IP 基础风控

## 评论状态设计

建议统一状态：

- `visible`
  正常展示
- `pending_review`
  等待审核
- `hidden`
  已隐藏
- `deleted`
  已删除

## 推荐审核流程

### 文本评论

1. 用户提交
2. 后端做基础校验
3. 敏感规则判断
4. 直接 `visible` 或进入 `pending_review`

### 带图评论

1. 用户上传图片
2. 图片先入存储
3. 评论创建时默认 `pending_review`
4. 审核通过后切为 `visible`

## 建议的规则层级

### L1 拒绝

直接拒绝，不入库或入库后立刻标记不可见：

- 明显广告
- 联系方式
- 重复垃圾文本
- 明显违规词

### L2 待审核

不直接放出：

- 带图评论
- 命中可疑词但不确定违规
- 新用户短时间连续发评论

### L3 直接放行

- 正常文本评论
- 正常长度
- 没有高风险特征

## 审核记录建议

建议新增 `moderation_events` 表，记录每次审核动作。

建议字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 记录 ID |
| `target_type` | text | `comment / comment_image / user` |
| `target_id` | uuid | 目标 ID |
| `action` | text | `auto_reject / auto_hold / approve / hide / delete` |
| `reason_code` | text | 原因码 |
| `reason_note` | text nullable | 人工备注 |
| `operator_user_id` | uuid nullable | 操作人，自动规则可为空 |
| `created_at` | timestamptz | 时间 |

## 反应类防刷建议

### 基础规则

1. 必须登录
2. 同一用户对同一点位只能有一个当前状态
3. 单用户短时间连续切换状态要限流
4. 异常频率的账号进入观察

### 推荐实现

- Redis 频控
- 数据库唯一约束
- 运营后台可按用户 / 点位查看异常操作量

## 评论频控建议

建议初版规则：

- 单用户单点位 30 秒内最多 1 条评论
- 单用户 10 分钟内评论总数有限制
- 图片上传数量有限制
- 单条评论图片数上限 9 张

## 展示策略建议

### 列表页和详情页

- 只展示 `visible` 评论
- `pending_review` 不公开显示
- 当前用户提交后，如果是 `pending_review`，可以在自己的视角看到“审核中”

## 第一版不做的能力

- 用户举报系统
- 审核工作台完整 UI
- 多级处罚系统
- 自动信誉分

这些都可以等评论量真正起来后再补。

## 当前推荐判断

第一版最稳的组合是：

- 文本评论：规则筛后尽量直接放
- 图片评论：默认待审核
- 点赞 / UP / 踩：不审核，但强制登录 + 频控
