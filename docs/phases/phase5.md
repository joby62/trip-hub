# Phase 5

## 目标

补齐景点视图、素材双归属和全站整合，把当前工程从“有完整功能的云南攻略页”收口成一套完整的 editorial trip 网站。

这一阶段重点解决：

- 景点维度浏览
- 图片与原文的双重组织
- 清单视图独立成立
- 搜索与深链升级
- 全站一致性与完整性校验

## 范围

- 新建景点视图
- 重构清单视图
- 建立 `day + attraction/theme` 双归属素材层
- 升级搜索与深链目标
- 做最后一轮视觉与内容收口

## Step By Step

1. 建立景点数据层
   从现有 docx 原文和图片中抽出核心景点实体，为每个景点建立基础资料、关联天数、关联段落和关联图片。

2. 给全部图片做双归属
   每张图至少要归属到一个 `day`，并且归属到一个 `attraction` 或 `theme`，不能再只按天散落。

3. 给原文段落做章节化归类
   在保留完整原文顺序的前提下，把段落补充映射到景点或计划节点，支持第二套浏览逻辑。

4. 新建景点视图
   参考 `stitch 2/attraction_detail/code.html`，为核心景点建立 Hero、费用、建议、图片、原文、避坑和预订入口。

5. 重做清单视图
   参考 `stitch 2/travel_checklist/code.html`，把打包清单、重要备忘和关键预订整合成独立视图。

6. 升级搜索
   搜索结果不只命中 `day / source / image / tool`，还应支持 `attraction`。

7. 升级深链
   在当前 `day / image / tool` 基础上，增加 `view / attraction` 级别的深链能力。

8. 做全量内容完整性校验
   核查没有任何图片遗漏，没有任何正文丢失，没有任何原文只能通过代码侧访问。

9. 做最终视觉收口
   统一各视图的 hero、card、glass、spacing、色板和排版层级，让整站风格一致。

## 交付物

- 景点视图
- 清单视图
- 全量图片双归属组织
- 全量原文章节化组织
- 升级后的搜索与深链
- 一致性的最终视觉系统

## 验收

- 用户既能按天浏览，也能按景点浏览。
- 所有图片都能被找到，并且至少有 `按天` 和 `按景点/主题` 两种组织路径。
- 文档正文完整可访问，不被摘要替代。
- 清单视图不再只是页面尾部模块，而是独立成立的工具页。
- 搜索和深链已经能覆盖新的景点维度。
