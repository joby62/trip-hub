# Phase 4

## 目标

在现有 `phase1-3` 的功能基础上，把页面从“单页功能堆栈”升级成更接近 stitch 2 参考稿的 editorial 旅行站骨架。

这一阶段不删任何功能，不删任何原文，不丢任何图片，重点是重做：

- 一级信息架构
- 一级导航逻辑
- 总览视图
- 行程视图
- 视觉层级与阅读顺序

## 范围

- 建立 `总览 / 行程 / 景点 / 清单` 四个一级视图骨架
- 重做顶部导航和底部导航
- 参考 `trip_overview` 重做总览视图
- 参考 `daily_itinerary` 重做行程视图
- 保留并迁移当前搜索、避坑、工具卡、原文、灯箱、深链能力

## Step By Step

1. 重组页面主容器
   把当前长页 section 结构重组为四个一级视图容器，而不是继续顺序堆叠。

2. 重做全站导航
   顶部导航改成 glass top bar，底部导航改为 `总览 / 行程 / 景点 / 清单`。

3. 重做总览 Hero
   参考 `stitch 2/trip_overview/code.html`，把第一屏改成大图主视觉 + 标题 + 时间 + 状态，不再以说明文字为主。

4. 重做总览区块顺序
   把当前 `overviewTools`、`pitfallFilters`、`bookingTools`、路线 spine 等内容，重排成更像旅行策展页的阅读顺序。

5. 将日程列表升级成章节式浏览
   不再让“摘要卡片 + 抽屉”承担主要阅读任务，而是让每一天成为可完整进入的章节。

6. 建立单日章节壳
   参考 `stitch 2/daily_itinerary/code.html`，给每一天建立更强的“时间流 / 计划流”容器。

7. 保留旧功能位并迁移位置
   搜索、避坑、图片、原文、工具卡、打包清单等不能消失，只能迁移到新的视图结构中。

8. 改第一轮视觉 token
   开始把页面从当前暖纸张功能卡风格，拉向 `jade_horizon` 的 teal / ochre / moss editorial 气质。

## 交付物

- 四个一级视图骨架
- 新的顶部 / 底部导航
- 新的总览视图
- 新的行程视图壳
- 旧功能在新结构中继续可用

## 验收

- 页面不再像“一个功能很全的长页”，而像旅行站首页和行程页。
- 搜索、避坑、灯箱、原文、工具卡、清单都还存在。
- 总览和行程的阅读顺序明显更接近 stitch 2 的 editorial 安排。
- 手机端仍然优先成立，桌面端只是增强。
