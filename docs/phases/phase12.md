# Phase 12

## 目标

做移动端最终整合与验收，把手机体验作为第一优先级正式收口。

这一阶段负责把前面三轮返工收成上线状态，并补齐断点、真机检查和残余不一致。

## 范围

- 全站移动端断点复核
- 真机尺寸下的视觉与交互校对
- 移动端性能、滚动、遮挡、抖动收口
- desktop / tablet 作为增强层重新对齐，不反向主导布局

## Step By Step

1. 做断点审计
   重点检查 `360 / 375 / 390 / 414 / 430` 等手机宽度下的导航、Hero、卡片、rail 和弹层。

2. 做真机体验清单
   检查顶部安全区、底部安全区、浏览器地址栏折叠、滚动回弹、键盘顶起和横竖屏切换。

3. 做性能收口
   优化首屏图片、滚动动画、灯箱切换和长页滚动时的体感问题。

4. 重新定义桌面增强边界
   desktop / tablet 只做展开式增强，不再反向决定移动端的基础结构。

5. 做最终移动端验收
   把“全部适配有问题”的问题单逐项关掉，形成明确的上线检查结果。

## 交付物

- 一套通过移动端验收的 trip guide
- 断点与真机检查记录
- 收口后的性能与交互一致性结果
- 明确的 desktop 增强边界

## 验收

- 手机端成为默认最佳体验，不再是 desktop 让位后的结果。
- 关键设备宽度下不再出现导航重叠、内容遮挡、图片失控和弹层难用问题。
- desktop 和 tablet 只做增强，不再主导布局策略。
