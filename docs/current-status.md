# Current Status

## 说明

这份文档是当前执行面板，回答的是：

1. 现在有哪些大块任务正在推进
2. 每条任务线现在卡在哪、下一步做什么
3. 哪些任务已经结项、是否进入归档

它和 `roadmap.md` 的区别是：

- `roadmap.md` 看长期方向和优先级
- `current-status.md` 看当前推进状态和结项状态

## 阻塞与风险

### 当前主要风险

1. 现有后端仍是静态壳，内容导入和社区互动都还没有真实服务能力。
2. 还缺部署、监控、环境变量和 worker 落地层面的运维文档。
3. 当前移动端页面还在快速变动，需要持续用状态板和时间流压住多人协作的发散。

## 更新规则

1. 某条任务进入实际开发时，把状态改成 `active`。
2. 某条任务达到阶段目标时，把状态改成 `done`。
3. 某条任务不再需要在主区持续跟进时，转移到归档并在 `timeline.md` 留结项记录。
4. 每次更新都要同步“最近更新”日期。
